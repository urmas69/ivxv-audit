# Parser and defensive-coding audit

The ZIP parser’s public entry point is `common/collector/zip/zip.go:143` (`EndOfCentralDirectory`), with bounded decompression at `86` and central-directory parsing at `504`; the repository includes regression tests for ZIP bombs (`zip_test.go:179`, `376-430`). These tests were executed through the audit harness; their result is preserved under `evidence/source-audit/test-results/`.

The admin command parser loads YAML at `collector-admin/ivxv_admin/command_file.py:390-424` and installs custom timestamp/container constructors. It calls `yaml.load(..., yaml.Loader)` at `408` and `424`; this is retained as an unresolved lead because the surrounding BDOC signature and filename checks (`141-243`) constrain reachability. A ten-second isolated Go fuzz attempt and static analysis were run; tool failures and outputs are recorded verbatim, not converted into security claims. Leads remain in [unresolved-leads.tsv](../evidence/source-audit/unresolved-leads.tsv).
