# Storage, administration, configuration and logging audit

`storage/service/storage/main.go:newCtrl` (`61-80`) parses storage configuration and configures TLS with `MinVersion: tls.VersionTLS12`. Startup writes the CA with mode `0600` (`276-281`), creates a Unix readiness socket (`283-290`), and starts `/usr/bin/etcd` (`293-307`). Errors from configuration, CA write, socket setup and logger construction return before normal service operation. The child process is intentionally not managed with `CommandContext` (`293`), making graceful-stop and crash-recovery behavior a deployment test requirement.

The hard-coded executable is confirmed finding `IVXV-SRC-001` (Medium robustness): relocation or a package layout without `/usr/bin/etcd` prevents storage startup. Missing artifact provenance remains in Phase 2 build/dependency reports, not as a source-code finding. `collector-admin/ivxv_admin/command_file.py:141-243` filters BDOC members and metadata, while `load_yaml_file` (`390-424`) uses custom constructors with `yaml.Loader`; this remains unresolved lead `LEAD-001` pending a signed-container reachability test. Event formatting is in `event_log.py:33-57`; correlation and access-control properties remain deployment assumptions.
