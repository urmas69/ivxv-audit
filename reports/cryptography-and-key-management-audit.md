# Cryptography and key management audit

The reviewed Java auditor and processor paths validate ciphertext/proof correspondence and consume configured key material. `auditor/tools/DecryptTool.java:107-179` checks proof entries against the anonymous ballot box and documents that the in-memory box is mutated while invalid/valid proof entries are removed; empty-box consistency is checked before completion. `auditor/tools/IntegrityTool.java:218-223` parses path components for log relationships. Go qualification implementations are `common/collector/q11n/ocsp/ocsp.go:49` and `q11n/tsp/tsp.go:83`; their log descriptors explicitly distinguish signature, OCSP and timestamp verification failures.

The fixed source review did not find a selectable weak algorithm or proof bypass. However, an algorithm/parameter inventory with exact constants and key-generation call sites remains incomplete until all `core/crypto` and Java crypto wrappers are traced. Certificate-chain, OCSP freshness, timestamp canonicalization, randomness and key-share lifetime require isolated malformed-certificate and key-service fixtures; they are not marked passed merely because library calls exist.
