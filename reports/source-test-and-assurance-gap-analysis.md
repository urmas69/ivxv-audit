# Source test and assurance gap analysis

The fixed tree contains Go unit tests and Java test source, but Phase 3 reconstruction reported Java Gradle test tasks as `NO-SOURCE`/`SKIPPED` and lacked election fixtures. The audit harness runs `go vet` and collector parser tests against the immutable checkout using the Phase 3 Go toolchain; raw logs are `evidence/source-audit/static-analysis/go-vet.log` and `evidence/source-audit/test-results/go-collector-tests.log`. A ten-second ZIP fuzz invocation is prepared and its exit status is in `evidence/source-audit/fuzzing/results.tsv`; the fixed checkout has no discoverable `Fuzz` function, so this documents a harness limitation rather than coverage.

The security-property map is [security-property-test-map.tsv](../evidence/source-audit/security-property-test-map.tsv). High-value gaps are revocation ordering, anonymization linkage absence, malformed signed containers, proof failure handling, retry/concurrency, certificate/OCSP failure and count-conservation. Java synthetic ballot-box fixtures are not present in the publication and are not fabricated as production evidence.
